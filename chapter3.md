# Capítulo III: Requirements Specification

## 3.1. To-Be Scenario Mapping.

En esta sección el equipo introduce y resume el proceso realizado para diseñar la experiencia futura ideal de los usuarios. Para la elaboración de estos mapas, el equipo siguió un proceso estructurado que consistió en las siguientes etapas:

1. **Preparación**: Establecimos los objetivos de la sesión y definimos claramente a nuestros User Personas ("Persona Ocupada" y "Aficionado").
2. **Lluvia de ideas individual**: Cada miembro del equipo aportó ideas de forma independiente sobre cómo debería ser la interacción ideal del usuario con la plataforma.
3. **Revisión e identificación de fases como columnas**: Agrupamos y ordenamos cronológicamente las ideas similares para definir el flujo paso a paso del usuario.
4. **Nombrar las fases**: Asignamos nombres representativos a cada columna para delimitar claramente las etapas de la experiencia.
5. **Comparar el mapa con el As-Is Scenario Mapping**: Contrastamos este nuevo mapa con nuestro mapa actual (As-Is) para evaluar la brecha entre la experiencia actual y la visión deseada.
6. **Identificando cambios que podría ofrecer el To-Be Scenario Mapping**: Con base en la comparación, extrajimos las mejoras clave y las nuevas características que el proyecto debe contemplar para aliviar los dolores de los usuarios.

A continuación, se presentan los To-Be Scenario Mapping elaborados para cada User Persona, estructurados con las fases identificadas y las filas fundamentales de **Doing**, **Thinking** y **Feeling**.

## 3.4. Impact Mapping.

### Segmento 1: Personas ocupadas (departamentos/oficinas)

Para el segmento de este usuario se elaboró un Impact Mapping con el objetivo de aumentar la participación activa en la plataforma y reducir la incertidumbre sobre el cuidado de las plantas. Esta herramienta nos permitió identificar los comportamientos clave que se desean promover en los usuarios, como consultar el estado de la humedad y configurar recordatorios automáticos de riego. A partir de ello, se definieron deliverables y user stories que guían el desarrollo de funcionalidades útiles y alineadas con las necesidades reales de los jóvenes universitarios que buscan mantener sus plantas sanas pese a su rutina ocupada.
